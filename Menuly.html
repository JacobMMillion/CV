Menuly: IOS application

Developed by Jacob Million

I created Menuly to help minimize an issue I noticed during my time working in the service industry: unorganization. Most of the time, menu descriptions are stored on something such as a google document which can lead to servers saving an outdated file or minor changes not being reflected on the document for long periods of time. The goal of Menuly is to allow managers to easily make changes to the menu descriptions, and to allow servers an easy interfance to referance and study.
Menuly works with a Firebase backend, both for the database for each restaurants menus and to store user login information. Every restaurant account is given a restaurant code, which non-admin accounts can sync with in order to get a live view of the menu. Users can update their code if they have multiple restaurant associations.
